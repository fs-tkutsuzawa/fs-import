# TODO: BS貸借一致の原則(G-01)の実装

## 概要

`bs_test_specifications.md`に定義されている最重要原則`【G-01】貸借一致の原則`を実装する。
これにより、FAMが生成するBSの会計的妥当性を保証する。

## TDDサイクル

### 1. RED: 貸借不一致で失敗するテストを追加

- BS計算後、`資産合計`と`負債・純資産合計`をそれぞれ計算する。
- `expect(assetsTotal).toBe(liabilitiesAndEquityTotal)`のようなアサーションを追加する。
- このテストは、貸借を一致させるロジックが未実装のため、必ず失敗する。

### 2. GREEN: 貸借を一致させる最小限のコードを実装

- `compute`メソッドの最終段階で、貸借差額を計算する。
- 差額を特定の勘定（通常は`現金(Cash)`）に加減算することで、強制的に貸借を一致させる（プラグ項目として扱う）。
- これにより、テストをパスさせる。

### 3. REFACTOR: 実装のクリーンアップ

- プラグとして現金を扱うロジックを、より明確なメソッドに切り出すなど、可読性と保守性を向上させる。
